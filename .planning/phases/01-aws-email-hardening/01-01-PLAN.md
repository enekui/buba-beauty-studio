  <action>
Ejecutar vía AWS CLI (fuera del módulo Terraform, porque el state no puede bootstrappearse a sí mismo):

```bash
ACCOUNT=372370374261
REGION=eu-west-1
BUCKET=buba-terraform-state-$ACCOUNT
TABLE=buba-terraform-locks

# Crear bucket
aws s3api create-bucket --bucket "$BUCKET" --region "$REGION" \
  --create-bucket-configuration LocationConstraint=$REGION

# Versioning
aws s3api put-bucket-versioning --bucket "$BUCKET" \
  --versioning-configuration Status=Enabled

# SSE-S3 (AES256)
aws s3api put-bucket-encryption --bucket "$BUCKET" --server-side-encryption-configuration \
  '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"AES256"},"BucketKeyEnabled":true}]}'

# Block public
aws s3api put-public-access-block --bucket "$BUCKET" --public-access-block-configuration \
  BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

# Lifecycle: expirar noncurrent versions a 90 días, borrar delete markers
aws s3api put-bucket-lifecycle-configuration --bucket "$BUCKET" --lifecycle-configuration '{
  "Rules":[{
    "ID":"expire-noncurrent","Status":"Enabled","Filter":{"Prefix":""},
    "NoncurrentVersionExpiration":{"NoncurrentDays":90}
  }]
}'

# Tags via put-bucket-tagging
aws s3api put-bucket-tagging --bucket "$BUCKET" --tagging \
  'TagSet=[{Key=Project,Value=buba},{Key=Environment,Value=prod},{Key=ManagedBy,Value=manual-bootstrap},{Key=Purpose,Value=terraform-state}]'

# DynamoDB table para locks
aws dynamodb create-table --table-name "$TABLE" --region "$REGION" \
  --billing-mode PAY_PER_REQUEST \
  --attribute-definitions AttributeName=LockID,AttributeType=S \
  --key-schema AttributeName=LockID,KeyType=HASH \
  --sse-specification Enabled=true \
  --tags Key=Project,Value=buba Key=Environment,Value=prod Key=Purpose,Value=terraform-lock

aws dynamodb wait table-exists --table-name "$TABLE" --region "$REGION"
```

Por qué NO gestionarlo con Terraform: bootstrap paradox (el bucket que guarda el state no puede estar *dentro* del state que va a guardar). Es estándar hacer esto con CLI o una cuenta separada de "meta" resources.
