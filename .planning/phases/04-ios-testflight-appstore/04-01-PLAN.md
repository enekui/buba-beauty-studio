<task type="auto">
  <name>Task 1: Upload .ipa a App Store Connect con xcrun altool</name>
  <files>—</files>
  <action>
App Store Connect requiere autenticación con un API key (alternativa: Apple ID + app-specific password). Preferir API key por seguridad.

**Prereq humano (si no existe):** ir a `https://appstoreconnect.apple.com/access/api`, crear una API key con role "App Manager", descargar el `.p8`, anotar Key ID + Issuer ID.

Upload via altool:

```bash
# Exportar credenciales (una vez por sesión). NO commitear.
export ASC_API_KEY_ID="ABCD123456"
export ASC_API_ISSUER_ID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
export ASC_API_KEY_PATH="$HOME/.appstoreconnect/AuthKey_${ASC_API_KEY_ID}.p8"

cd ios-app/ios
xcrun altool --upload-app -f build/App.ipa -t ios \
  --apiKey "$ASC_API_KEY_ID" \
  --apiIssuer "$ASC_API_ISSUER_ID" 2>&1 | tail -20
```

Tras el upload, App Store Connect tarda 5-15 min en procesar el binario. Durante este tiempo el estado pasa de "Waiting for Review" a "Processing" a "Ready to Test".

**Alternativa si no se quiere API key**: `xcrun altool --upload-app -u <apple-id> -p <app-specific-password>` (requiere generar un app-specific password en appleid.apple.com).
  </action>
  <verify>
```bash
xcrun altool --list-apps --apiKey "$ASC_API_KEY_ID" --apiIssuer "$ASC_API_ISSUER_ID" | grep -i "buba"
```
Debe aparecer la app.
  </verify>
  <done>Upload exitoso (altool devuelve "Package Summary: No errors" o equivalente).</done>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Build subida a App Store Connect, pendiente de asignación a TestFlight.</what-built>
  <how-to-verify>
1. En `https://appstoreconnect.apple.com/apps`, seleccionar "Buba Beauty Studio".
2. Tab **TestFlight** → la build debe aparecer con estado "Ready to Test" (tras procesarse). Puede pedir completar "Export Compliance" (declarar si la app usa encryption — responder "No" si no hay custom crypto).
3. Click en la build → sección **Internal Testing** → crear grupo "Salón" si no existe → añadir testers:
   - Tu Apple ID (adianny@me.com).
   - Apple ID de la dueña del salón.
4. Apple envía email a los testers con invitación TestFlight + link para instalar la app TestFlight si no la tienen.
5. En el iPhone físico, aceptar invitación → instalar build.
6. Abrir la app. Smoke test:
   - Splash cream con logo → landing.
   - Scroll, galería, reseñas OK.
   - Tap "Reservar ahora" → Booksy embebido.
   - Tap Llamar → diálogo nativo `tel:`.
   - Tap WhatsApp → abre WhatsApp (instalado en el iPhone).
   - Tap Maps → Apple Maps con Rúa Curtis 8.
   - App pide permiso de notificaciones al primer arranque. Aceptar.
7. Desde OneSignal dashboard → **Messages** → New Push → Send to subscribed users → debería recibirse en el iPhone en <30s.

Screenshot(s) de la app instalada + push recibida en `.planning/phases/04-ios-testflight-appstore/evidence/`.
