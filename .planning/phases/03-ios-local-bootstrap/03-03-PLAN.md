<task type="checkpoint:human-action" gate="blocking">
  <action>Crear app en OneSignal + generar APNs .p8 en Apple Dev</action>
  <instructions>
1. Ir a `https://dashboard.onesignal.com` y crear cuenta si no existe (gratis).
2. **New App/Website** → nombre: `Buba Beauty Studio` → platform: Apple iOS (APNs).
3. Copiar el **App ID** que OneSignal asigna (UUID).
4. OneSignal pide el **APNs Auth Key (.p8)**. Para generarla:
   - `https://developer.apple.com/account/resources/authkeys/list` → botón `+` → Keys → name: `Buba Beauty APNs Key` → marcar checkbox **Apple Push Notifications service (APNs)** → Continue → Register → **Download** el `.p8` (solo se puede descargar UNA vez; guardarlo bien).
   - Anotar también el **Key ID** (10 chars) y el **Team ID** del step anterior.
5. Volver a OneSignal y pegar: Key ID, Team ID, subir el archivo .p8, Bundle ID `com.bubabeautystudio.app`.
6. Save. OneSignal debería mostrar "Active" en la plataforma iOS.
  </instructions>
  <verification>OneSignal dashboard muestra la app con iOS platform "Active". Team ID + Key ID + .p8 subidos.</verification>
  <resume-signal>Escribir "OneSignal configurado, App ID=<uuid>".</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Sustituir ONESIGNAL_APP_ID en js/push.js y re-sync</name>
  <files>js/push.js, ios-app/www/js/push.js</files>
  <action>
1. Editar `js/push.js` y reemplazar el literal `REPLACE_WITH_ONESIGNAL_APP_ID` con el App ID real devuelto por OneSignal.
2. `cd ios-app && npm run build:web && npx cap sync ios` para que el `www/` embebido en el bundle iOS se regenere con la nueva URL.
3. Commit y push (recordar: en este repo permitido commit grande).

Evitar: NO commitear el archivo `.p8` ni Key ID / Team ID en texto plano en el repo (ambos son secretos aunque el Key ID sea menos crítico). OneSignal los guarda; no hacen falta en el código cliente.
  </action>
  <verify>`grep ONESIGNAL_APP_ID js/push.js | grep -v REPLACE_WITH` debe matchear el nuevo UUID.</verify>
  <done>js/push.js apunta al App ID real, ios-app/www/js/push.js está en sync, commit pusheado.</done>
</task>

<task type="auto">
  <name>Task 3: Archive firmado vía xcodebuild</name>
  <files>ios-app/ios/build/App.xcarchive, ios-app/ios/build/App.ipa</files>
  <action>
```bash
cd ios-app/ios
xcodebuild -workspace App/App.xcworkspace -scheme App \
  -configuration Release \
  -destination "generic/platform=iOS" \
  -archivePath build/App.xcarchive \
  archive | tail -20
```

Si el archive tiene éxito, exportar el `.ipa`:

```bash
cat > /tmp/exportOptions.plist <<'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>method</key>
    <string>app-store-connect</string>
    <key>signingStyle</key>
    <string>automatic</string>
    <key>uploadSymbols</key>
    <true/>
</dict>
</plist>
EOF

xcodebuild -exportArchive -archivePath build/App.xcarchive \
  -exportPath build/ \
  -exportOptionsPlist /tmp/exportOptions.plist | tail -10
```
  </action>
  <verify>
```bash
test -f ios-app/ios/build/App.xcarchive/Info.plist
test -f ios-app/ios/build/App.ipa
```
  </verify>
  <done>`.xcarchive` y `.ipa` existen. Tamaño del .ipa razonable (>5 MB, <100 MB).</done>
</task>
